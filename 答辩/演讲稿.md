# 01_封面

今天，我要向你们展示一个东西。一个能让旅游信息查找变得简单到不可思议的东西。

# 02_痛点画面

你们有没有过这样的经历？

你打开手机，想查一下去贵州玩什么。携程上有一份攻略，马蜂窝上有一份，小红书上还有十几篇。你想知道黄果树瀑布门票多少钱——对不起，你得翻五个页面。你想知道怎么去——对不起，再去另外三个App查。

一个简单的旅行计划，你要在十几个平台之间来回跳转。最后你拼凑出来的信息，还不知道哪个是对的。

这太荒谬了。信息明明都在那里，但你就是找不到。

# 03_产品定位

所以我们做了一个东西。

你只需要问一个问题。随便问，用你自己的话。比如——"黄果树瀑布门票多少钱"。

然后，两秒钟。

答案就出来了。而且它会告诉你，这个答案是从哪里来的。不是瞎编的，是有据可查的。

就这么简单。输入问题，得到答案。

# 04_核心功能

它能做什么？三件事。

第一，智能问答。你问什么，它答什么。门票、交通、开放时间、推荐——它都能理解，都能回答。

第二，景点地图。117个贵州景点，全部标注在地图上。你可以按类别筛选，可以看路线，可以看图片。

第三，信息管理。所有的景点、攻略、路线，都可以在后台管理。增删改查，一个不少。

# 05_系统架构

你们可能会问——这背后是什么？

四层架构。前端是HTML、CSS、JavaScript加上Leaflet（一个开源的JavaScript地图库，能在网页上显示可交互的地图，支持缩放、拖动、标注）。后端是Python Flask（Python的一个轻量级Web框架，负责接收前端请求、处理业务逻辑、调用AI模块、返回结果）。中间是AI核心——这是最关键的部分。最底层是SQLite数据库（一种轻量级嵌入式数据库，不需要单独安装数据库服务器，整个数据库就是一个.db文件），存着117个景点的全部信息。

每一层都在做它该做的事。简单、清晰、高效。

# 06_RAG流程

AI核心是怎么工作的？六个步骤。

你输入问题。系统先判断你问的是什么——门票？交通？还是推荐？然后它去数据库里搜索，用TF-IDF（词频-逆文档频率，一种文本匹配算法，通过计算一个词在当前问题中出现的频率和在所有文档中出现的稀有程度，来判断哪些景点与你的问题最相关）找到最相关的景点。最后，它把这些信息整合起来，给你一个完整的答案。

整个过程，基于真实的数据。不会胡说八道，不会凭空捏造。每个回答，都有来源。

# 07_项目成果

用数字说话。

117个景点。5种意图分类。6张数据表。响应时间——不到一秒。

这些数字意味着什么？意味着这是一个真正能用的系统。不是demo，不是玩具，是一个完整的产品。

# 08_开发过程

我们是怎么做到的？五个阶段。

先搞清楚用户到底需要什么。然后去爬数据——117个景点，从多个来源整合。接着设计系统架构。然后写代码——前后端一起上。最后测试、优化、打磨。

技术栈？Python、Flask、SQLite。前端HTML、CSS、JavaScript。AI核心是TF-IDF检索加RAG（检索增强生成，一种让AI先从数据库中检索相关资料，再基于这些真实资料生成回答的技术，避免AI凭空捏造信息）增强。不花哨，但管用。
